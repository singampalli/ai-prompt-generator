Example: Making a Clear Decision

Use this when you need help choosing between options, not just understanding them.

  • Task → Ask for a decision
  • Context → Add limits (time, cost, capacity)
  • Audience → Shapes tone
  • Persona → Ensures balanced thinking
  • Precision → Forces recommendation
Task:
Decide whether to keep improving Jenkins
or move to managed CI/CD on GCP.

Context:
Limited ops capacity and cost sensitivity.

Audience:
Engineering Manager

Persona:
Solution Architect

Precision Level:
Decision-making analysis

Result: Clear trade-offs and a recommendation.
